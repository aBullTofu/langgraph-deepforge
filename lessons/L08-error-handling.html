Phase 1 · GodView

🗺 上帝视角

🧠 四层防御体系

层级策略LangGraph 实现
1. 瞬时故障自动重试，指数退避RetryPolicy(max_attempts=3, backoff_factor=2)
2. LLM 感知工具返回错误信息给 LLMToolMessage("错误：{msg}，请调整重试")
3. 人工介入暂停，等人类决定interrupt() 在重试耗尽后
4. 降级回复安全停止，记录日志try/except → fallback 回复
Phase 2 · Combat

⚔️ 实战编码 ⏱ 25 分钟

第 1 层：RetryPolicy

from langgraph.types import RetryPolicy
builder.add_node("api_call", call_external_api, retry_policy=RetryPolicy(
    max_attempts=3, initial_interval=1.0, backoff_factor=2.0
))  # 重试间隔: 1s → 2s → 4s

第 2 层：LLM 感知错误（手动 act() 中）

try: result = tool.invoke(args)
except Exception as e:
    result = f"错误：{e}。请调整参数或换种方式重试。"  # LLM 看到后自行调整

第 3+4 层：组合使用

# RetryPolicy 自动重试 3 次 → 还失败 → interrupt 人工决定 → 降级
builder.add_node("critical_action", critical_action, retry_policy=RetryPolicy(max_attempts=3))
⚠️ 避坑：RetryPolicy 只对瞬时故障（网络超时）有效。如果 API 返回"参数错误"，重试 100 次也没用——应该走第 2 层（错误传给 LLM 让它改参数）。
