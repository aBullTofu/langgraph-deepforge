📖 寓言：桥上的收费站

📖 Fable: The Tollbooth on the Bridge

有一座桥，桥中间设了收费站。不是每辆车都要停——空车直接过，普通货车自动扣费，但运输火药、运送囚犯、装载金银的车必须在收费站停下，由守桥官亲自检查货物清单，签字盖章后才能放行。有人问为什么不自动通过所有车？守桥官说："三个月前，一辆声称'运输粮食'的车过桥时爆炸了。"

🔮 揭示：interrupt() 就是收费站栏杆——敏感操作暂停，等人签字才放行。

Phase 2 · Combat

⚔️ 实战编码 ⏱ 30 分钟30 min

from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import InMemorySaver  # ⚠️ interrupt 必须配合 checkpointer！# interrupt requires a checkpointer!

def dangerous_action(state) -> dict:
    # ⭐ 在这里暂停，等待人类确认# Pause here, wait for human confirmation
    approved = interrupt({"message": "即将删除数据库记录，确认？"})
    if not approved: return {"result": "已取消"}
    return {"result": "已执行"}

# 第一次调用：运行到 interrupt() 暂停# First call: runs to interrupt() and pauses
config = {"configurable": {"thread_id": "1"}}
result = graph.invoke(input, config)
if "__interrupt__" in result:
    print(f"⏸️ {result['__interrupt__']}")
    decision = input("批准？(y/n): ")
    # 第二次调用：恢复执行# Second call: resumes execution
    result = graph.invoke(Command(resume=(decision == "y")), config)
⚠️ 避坑：interrupt() 必须配合 checkpointer。没有 checkpointer → 行为不可预测。进程重启后能否恢复取决于 checkpointer 类型（InMemory 不行，Sqlite/Postgres 可以）。
⚠️ Pitfall: interrupt() must be paired with a checkpointer. Without a checkpointer, behavior is unpredictable. Recovery after process restart depends on the checkpointer type: InMemory cannot recover; SQLite/Postgres can.
