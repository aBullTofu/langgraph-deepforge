Phase 1 · GodView

🗺 上帝视角 ⏱ 7 分钟7 min

📋 通关条件

📋 Clear Condition

构建能真正调用函数的 Agent：用户说"3+5"，Agent 调用 add 工具计算，返回精确结果。

🧠 心智模型：Tool = 函数 + 说明书

🧠 Mental Model: Tool = Function + Manual

@tool 装饰器做了两件事：① 保留函数可执行 ② 自动生成一段 JSON Schema（工具名、描述、参数类型）嵌入 LLM 请求中。LLM 看到说明书后决定"要不要调、调哪个、传什么参数"。LLM 不执行工具——它只输出一个 tool_calls 列表，由 ToolNode 执行。

# @tool 等价于给函数贴了一张"说明书"给 LLM 看
@tool
def add(a: int, b: int) -> int:
    """加法。两个整数相加。"""
    return a + b

Step 1：定义工具 + 绑定到模型

Step 1: Define Tools + Bind to Model

from langchain.tools import tool
from langgraph.prebuilt import ToolNode
from langgraph.graph.message import add_messages
from langchain.messages import AnyMessage, SystemMessage
import operator

@tool
def add(a: int, b: int) -> int: """加法运算"""; return a + b
@tool
def multiply(a: int, b: int) -> int: """乘法运算"""; return a * b
@tool
def get_weather(city: str) -> str: """查询城市天气"""
    data = {"北京": "晴 25°C", "上海": "多云 28°C"}
    return data.get(city, f"未找到{city}")

tools = [add, multiply, get_weather]
model = init_chat_model("deepseek-chat")
model_with_tools = model.bind_tools(tools)  # ⭐ 关键：把工具说明书传给 LLM# Key: pass the tool manual to the LLM

Step 2：定义 State（⚠️ 必须用 add_messages reducer！）

Step 2: Define State (⚠️ Must use add_messages reducer!)

class AgentState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]  # ⭐ 追加而非覆盖！
    # 如果用普通的 list，ToolNode 返回的 ToolMessage 会覆盖整个对话历史
    # → 导致 OpenAI 400 错误：tool role 前缺少 tool_calls

Step 3：LLM 节点 + 条件边

Step 3：LLM Node + Conditional Edge

def llm_node(state: AgentState) -> dict:
    msgs = [SystemMessage("你是助手，需要计算时调用工具")] + state["messages"]
    return {"messages": [model_with_tools.invoke(msgs)]}

tool_node = ToolNode(tools)  # 预构建节点，自动解析 tool_calls 并执行# Pre-built node, auto-parses tool_calls and executes

def should_call_tool(state: AgentState) -> str:
    last = state["messages"][-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"
    return END

Step 4：组装（LLM ↔ Tools 循环）

Step 4: Assemble (LLM ↔ Tools Loop)

builder = StateGraph(AgentState)
builder.add_node("llm", llm_node); builder.add_node("tools", tool_node)
builder.add_edge(START, "llm")
builder.add_conditional_edges("llm", should_call_tool, {"tools": "tools", END: END})
builder.add_edge("tools", "llm")  # 工具结果返回 LLM，形成循环
graph = builder.compile()

Step 5：测试

Step 5: Test

for q in ["3+5=?", "7*8=?", "北京天气？", "你好"]:
    r = graph.invoke({"messages": [HumanMessage(q)]})
    print(f"🤖: {r['messages'][-1].content}")

⚠️ Pitfall: INVALID_CHAT_HISTORY This is the #1 community error. Cause: messages in State does not use the add_messages reducer → ToolNode output overwrites the LLM output → conversation history is missing. See Step 2 above for the fix.
