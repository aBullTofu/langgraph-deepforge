📋 通关条件

📋 Clear Condition

构建意图路由器：用户输入一句话 → DeepSeek 判断意图（闲聊/查资料/投诉）→ 走不同分支处理。

本课新概念（L01 没出现过的）

init_chat_model()
之前你要手写 HTTP 请求调 API，这个函数一行搞定。跟 L01 的 StateGraph() 一样，背后藏了复杂逻辑，你只需要知道它返回一个能调 LLM 的对象。

model.invoke(prompt)
把文本发给 LLM 并拿回结果。跟 L01 的 graph.invoke() 是同一个动词 invoke，不是巧合。LangGraph 统一了这个调用习惯，以后你会反复见到。

条件边 + LLM
L01 的条件边是你写的 if count < 20，规则是死的。L02 让 LLM 读用户输入，自己判断走哪条路。条件边还是那条条件边，但路由决策从"你定的"变成"LLM 定的"。这意味着你需要处理 LLM 可能犯的错。

prompt 工程
不是写 Python 代码，是写一段自然语言告诉 LLM "只准返回这三个词之一，不要解释"。prompt 写得好不好，直接影响路由准确率。你在 Step 2 里会看到第一个 prompt。

else 兜底
LLM 不听话。你让它返回 "chat"，它可能返回 "Chat"、"闲聊"、甚至带换行符。route 函数的末尾如果不写 else: return "handle_chat"，任何意外值都会让图静默停止。多写一行 else，少踩一天坑。

New Concepts (not covered in L01)

init_chat_model()
Previously you'd hand-write HTTP requests to call an API. This function does it in one line. Like L01's StateGraph(), it hides complexity behind a simple constructor. You just need to know it returns an object that can call an LLM.

model.invoke(prompt)
Sends text to the LLM and gets a result back. Same verb invoke as L01's graph.invoke() — not a coincidence. LangGraph standardized on this calling convention, and you'll see it everywhere.

Conditional Edge + LLM
L01's conditional edge ran your if count < 20 rule. L02 lets the LLM read user input and decide which path to take. Same conditional edge mechanism, but now the routing decision comes from the LLM instead of you. That means you need to handle the LLM's mistakes.

Prompt Engineering
Not writing Python code. It's writing natural language telling the LLM "only return one of these three words, no explanations." How well you write the prompt directly affects routing accuracy. You'll see your first prompt in Step 2.

Else Fallback
The LLM doesn't always follow instructions. You ask for "chat" and it might return "Chat", Chinese characters, or a line break. If your route function doesn't have an else: return "handle_chat" at the end, any unexpected return value makes the graph silently stop. One extra line of else saves a day of debugging.

Step 2：定义 State + LLM 分类节点

Step 2: Define State + LLM Classification Node

class RouterState(TypedDict):
    user_input: str
    intent: str
    response: str

def classify_intent(state: RouterState) -> dict:
    prompt = f"""分析用户输入，只返回一个标签：chat(闲聊)、search(查询)、complaint(投诉)。
用户输入：{state["user_input"]}
标签："""
    result = model.invoke(prompt)
    return {"intent": result.content.strip().lower()}

Step 3：三个处理分支 + 条件边

Step 3：Three processing branches + Conditional Edge

def handle_chat(state): return {"response": f"🤝 {state['user_input']}——有什么可以帮你的？"}
def handle_search(state): return {"response": f"📚 关于「{state['user_input']}」，建议查阅官方文档。"}
def handle_complaint(state): return {"response": f"🚨 已收到反馈，客服将联系您。"}

def route_by_intent(state: RouterState) -> str:
    intent = state["intent"]
    if intent == "chat": return "handle_chat"
    elif intent == "search": return "handle_search"
    elif intent == "complaint": return "handle_complaint"
    else: return "handle_chat"  # ⭐ LLM 返回意外值时的兜底

Step 4：组装 + 测试

Step 4：Assembly + Test

builder = StateGraph(RouterState)
builder.add_node("classify", classify_intent)
builder.add_node("handle_chat", handle_chat)
builder.add_node("handle_search", handle_search)
builder.add_node("handle_complaint", handle_complaint)
builder.add_edge(START, "classify")
builder.add_conditional_edges("classify", route_by_intent, {"handle_chat": "handle_chat", "handle_search": "handle_search", "handle_complaint": "handle_complaint"})
builder.add_edge("handle_chat", END); builder.add_edge("handle_search", END); builder.add_edge("handle_complaint", END)
graph = builder.compile()

tests = ["你好呀", "Python怎么读CSV", "我要退款"]
for t in tests:
    r = graph.invoke({"user_input": t})
    print(f"👤 {t}\n🎯 {r['intent']}\n🤖 {r['response']}\n")

⚠️ 避坑：LLM 返回意外值

⚠️ Pitfall：LLM Returns unexpected values

常见问题：LLM 返回了 "Chat"（大写）或 "闲聊"（中文），条件边映射字典里没有 → 图静默停止。解决：① prompt 中强调格式 ② router 函数中加 else 兜底 ③ 调用 .strip().lower() 做归一化。
Common issue: The LLM returns "Chat" (uppercase) or "chat_zh" (unsupported label), which is missing from the conditional-edge mapping dictionary, so the graph stops silently. Fix it by: ① emphasizing the output format in the prompt, ② adding an else fallback in the router, and ③ calling .strip().lower() for normalization.

The LLM returns a string like '{"intent":"chat","confidence":0.95}'. Use data = json.loads(result.content) to turn it into a Python dict, then access data["intent"] and data["confidence"]. Note: the LLM sometimes wraps JSON in backticks or adds commentary. Call .strip("`") first to clean it up. If json.loads throws, the LLM didn't follow the format — use try/except as a safety net.

import json
def classify_intent(state):
    prompt = """返回 JSON: {"intent":"chat/search/complaint/order","confidence":0.0~1.0}。低置信度表示不确定。只返回 JSON，不要其他文字。"""
    result = model.invoke(prompt)

        data = {"intent": "chat", "confidence": 0}
    return {"intent": data["intent"], "confidence": data["confidence"]}
def route_by_confidence(state):

Q1 (L1-Q1) LangGraph 和 LangChain 的区别？什么时候用哪个？

Q1 (L1-Q1) What's the difference between LangGraph and LangChain? When to use which?

点击查看答案Click to reveal

LangChain 是 LLM 工具链（链式调用、RAG、prompt 模板）；LangGraph 是状态图引擎（多步推理、分支循环、条件路由）。LangChain 解决"怎么调 LLM"，LangGraph 解决"LLM 之间怎么协作"。简单场景用 LangChain，复杂 Agent + 多步决策用 LangGraph。LangChain is an LLM toolkit (chains, RAG, prompt templates); LangGraph is a state graph engine (multi-step reasoning, branching, looping, conditional routing). LangChain solves "how to call an LLM"; LangGraph solves "how LLMs collaborate." Use LangChain for simple scenarios, LangGraph for complex agents and multi-step decision-making.

Q2 (L2-Q3) 条件边函数返回了映射字典中没有的值会怎样？

Q2 (L2-Q3) What happens if a conditional edge returns a value not in the mapping?

点击查看答案Click to reveal

图会静默停止——不报错，不输出，只是不动了。这是 LangGraph 最大的调试陷阱之一。解决方案：① route 函数必须有 else 兜底 ② 用 .strip().lower() 归一化 LLM 输出 ③ 在 prompt 中约束输出格式。The graph stops silently — no error, no output, just freezes. This is one of LangGraph's biggest debugging traps. Fix: ① always include an else fallback in route functions, ② normalize LLM output with .strip().lower(), ③ constrain output format in the prompt.

Q3 (L2-Q5) Agent 陷入死循环怎么办？

Q3 (L2-Q5) How to prevent an agent from getting stuck in a loop?

点击查看答案Click to reveal

三层防线：① 在 compile(recursion_limit=N) 设置硬上限（默认 25）② 在 State 中维护计数，条件边判断超限则强制 END ③ prompt 工程——明确告诉 LLM "当你得到答案后直接回复用户，不要再调工具"。Three layers of defense: ① Set a hard limit with compile(recursion_limit=N) (default 25), ② maintain a counter in State — the conditional edge forces END when the limit is exceeded, ③ prompt engineering — explicitly tell the LLM "when you have the answer, reply directly without calling more tools."
