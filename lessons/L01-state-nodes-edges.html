Phase 2 · Combat

⚔️ 实战编码 ⏱ 30 分钟30 min

创建 l01_pipeline.py，跟着写：

Step 1：定义 State（物料篮）

Step 1: Define State (Material Basket)

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class PipelineState(TypedDict):
    text: str            # 原始输入# original input
    word_count: int      # 字数统计结果# character count result
    category: str        # short / long
    result: str          # 最终处理结果# final processing result

Step 2：计数工位

Step 2: Count Workstation

def count_words(state: PipelineState) -> dict:
    """节点函数的铁律：
    1. 第一个参数是 state（LangGraph 自动传入）
    2. 返回 dict，只包含要更新的字段
    3. 不需要返回所有字段——LangGraph 自动合并
    """
    text = state["text"]
    return {"word_count": len(text)}

Step 3：分类工位（质检台）

Step 3: Classification Workstation (Inspector)

def classify(state: PipelineState) -> dict:
    count = state["word_count"]  # 读取上一个节点写入的数据
    if count < 20:
        return {"category": "short"}
    else:
        return {"category": "long"}

Step 4：两个处理分支

Step 4: Two Processing Branches

def handle_short(state: PipelineState) -> dict:
    return {"result": f"[快速通道] {state['text']}"}

def handle_long(state: PipelineState) -> dict:
    return {"result": f"[深度处理] {state['text'][:20]}..."}

Step 5：⭐ 条件边函数（传送带分岔器）

Step 5: ⭐ Conditional Edge Function (Conveyor Diverter)

# 条件边函数的签名：(state) → str
# 返回值必须是 add_node 时注册的名字# Return value must be the name registered during add_node
def route_by_length(state: PipelineState) -> str:
    if state["category"] == "short":
        return "handle_short"
    else:
        return "handle_long"

Step 6：组装图

Step 6: Assemble the Graph

builder = StateGraph(PipelineState)

builder.add_node("count_words", count_words)
builder.add_node("classify", classify)
builder.add_node("handle_short", handle_short)
builder.add_node("handle_long", handle_long)

# 固定边# Fixed edge
builder.add_edge(START, "count_words")
builder.add_edge("count_words", "classify")

# ⭐ 条件边 — 本课核心# Conditional edge — core of this lesson
builder.add_conditional_edges(
    "classify",         # 从哪个节点出发# Which node to start from
    route_by_length,      # 路由函数# Routing function
    {                     # 返回值 → 目标节点 映射# Return value → target node mapping
        "handle_short": "handle_short",
        "handle_long": "handle_long",
    }
)

builder.add_edge("handle_short", END)
builder.add_edge("handle_long", END)

graph = builder.compile()

Step 7：运行测试

Step 7: Run Tests

# 短文本# Short text
r1 = graph.invoke({"text": "你好"})
print(f"字数: {r1['word_count']}, 分类: {r1['category']}, 结果: {r1['result']}")

# 长文本# Long text
r2 = graph.invoke({"text": "今天天气真好我们去公园散步吧顺便买个冰淇淋吃"})
print(f"字数: {r2['word_count']}, 分类: {r2['category']}, 结果: {r2['result']}")

🔍 模式 B：Debug 挑战

🔍 Mode B: Debug Challenge

下面这段代码有 3 个 bug。找出并修复它们：

class State(TypedDict):
    value: str

def step_a(state):
    return {"value": state.value.upper()}  # bug 1

def step_b(state: State) -> dict:
    return {"result": f"Done: {state['value']}"}  # bug 2

builder = StateGraph(State)
builder.add_node("a", step_a)
builder.add_node("b", step_b)
builder.add_edge(START, "b")  # bug 3
builder.add_edge("a", END)
graph = builder.compile()
点击查看答案Click to reveal answer

Bug 1：state.value 应该是 state["value"] — State 是 dict-like，不是对象属性。

Bug 2：返回了 "result" 字段，但 State 中没有定义这个字段 — 要么先在 State 中声明 result: str，要么返回已有字段。

Bug 3：边连接了 START → b，但 a 被孤立了（没有入口）。应该 START → a，a → b，b → END。
