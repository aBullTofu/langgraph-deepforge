Phase 1 · Fable

📖 寓言：书记官的账本

古罗马元老院有一规矩：每一场辩论，书记官必须在每个人发言结束后立即把发言记在羊皮纸上。如果辩论被地震打断，下次开会翻开羊皮纸，从断点继续。千年之后，一个叫 LangGraph 的程序继承了书记官的智慧：每一个 super-step 都是一条新记录。

🔮 揭示：Checkpoint 持久化。每个 super-step 自动存档，用 thread_id 区分不同"辩论"。

📋 通关条件

给 L04 的 Agent 加上持久化记忆——对话结束后 Agent 记住上下文，下次对话继续。

🧠 心智模型：Checkpoint = 游戏存档

Phase 2 · Combat

⚔️ 实战编码 ⏱ 30 分钟

创建 l05_memory_agent.py。改 L04 的代码只需三行：

# 1. 创建 checkpointer
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.checkpoint.sqlite import SqliteSaver
import sqlite3
checkpointer = InMemorySaver()  # 开发用（重启丢失）
# 或 checkpointer = SqliteSaver(sqlite3.connect("agent.db"))  # 持久化

# 2. 编译时注入
graph = builder.compile(checkpointer=checkpointer)  # 改这一行

# 3. invoke 时传 config（改这一行）
config = {"configurable": {"thread_id": "user-bob"}}
result = graph.invoke(input, config)  # 同一 thread_id = 同一会话

时间旅行

snapshot = graph.get_state(config)  # 最新状态
history = list(graph.get_state_history(config))  # 全部历史
# 回到某个历史检查点
old = history[3].config["configurable"]["checkpoint_id"]
old_state = graph.get_state({"configurable": {"thread_id": "user-bob", "checkpoint_id": old}})

记忆测试

graph.invoke({"messages": [HumanMessage("我叫小明")]}, config)
r = graph.invoke({"messages": [HumanMessage("我叫什么？")]}, config)
print(r["messages"][-1].content)  # "你叫小明！" ✅ 记住了
⚠️ 避坑 MISSING_CHECKPOINTER：用了 interrupt() 或想用 get_state() 但没配置 checkpointer → 报错。每个需要持久化的图都必须 compile(checkpointer=...)。
