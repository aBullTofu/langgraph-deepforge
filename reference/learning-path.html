🗺 学习路线图

从零基础到能独立构建生产级 LangGraph Agent 系统 — 三阶段 · 12 课 · 100 道面试题

预计 6-8 周 3 个阶段里程碑 12 个通关检查点

📐 整体框架

阶段课程学习目标面试对应层建议时间
🟢 基础篇 L01 — L04 能独立构建单 Agent 应用 L1 基础层 2 周
🟡 专业篇 L05 — L08 能构建生产级 Agent（持久化+流式输出+人机协同） L3 核心层+L4 高级层 2-3 周
🔴 专家篇 L09 — L12 能设计多 Agent 架构并部署到生产 L5 架构层 2-3 周

🔧 起点：你需要什么

硬性前置条件（必须）

  • Python 3.10+ — 能写函数、类、装饰器、async/await
  • 理解基本的 HTTP API 调用（requests / httpx）
  • 拥有至少一个 LLM API Key（DeepSeek / OpenAI / Anthropic 任一）

软性前置（有帮助但不是必须）

  • 了解 Transformer 的基本概念（Self-Attention 做了什么，不用推公式）
  • 用过至少一次 LangChain 的 `ChatOpenAI` 或类似 LLM 封装
  • 听说过 RAG（检索增强生成）的概念

🟢 第一阶段：构建你的第一个 Agent（L01-L04）

目标：理解 LangGraph 的核心抽象，能独立写出一个带工具调用的 Agent 循环。

L01 · State / Node / Edge 基石

  • 核心概念：StateGraph, State (TypedDict), Node, Edge, Reducer (add_messages)
  • 必做练习：用 StateGraph 写一个 3 节点的线性图（输入→处理→输出）
  • 寓言语境：水渠与磨坊 — State 是水流，Node 是磨坊，Edge 是水渠
  • 完成后自检：能用 3 句话解释 State/Node/Edge 给一个非技术朋友听
  • 避坑：add_messages 缺失导致消息覆盖（社区 #1 报错）
  • 面试预热：L3 面试题 Q36-Q40 — StateGraph 基础
    • ↓

    L02 · 第一个 LLM Agent Agent

  • 核心概念：LLM 节点、条件边（tools_condition）、第一个 Agent 循环
  • 必做练习：构建一个能回答"今天天气怎么样"并调用天气工具的 Agent
  • 寓言语境：十字路口的客栈 — 条件边是十字路口，LLM 决定走哪条
  • 避坑：条件边返回值不在 path_map 中 → 图静默退出
  • 面试预热：Q41-Q42 — 条件路由 + tools_condition 调试
    • ↓

    L03 · Tools（工具集成）工具

  • 核心概念：@tool 装饰器、ToolNode、bind_tools、ToolMessage、MCP 协议入门
  • 必做练习：设计 3 个工具（搜索、计算器、数据库查询）并集成到 Agent
  • 寓言语境：学徒的工具箱 — 每个工具是学徒掌握的专用技能
  • 避坑：工具返回异常 → 给 LLM 返回描述性错误消息而不是抛异常
  • 面试预热：Q29-Q30 — create_agent + MCP 集成
    • ↓

    L04 · Agent Loop（ReAct 循环）循环

  • 核心概念：手写 ReAct（Think→Act→Observe→Repeat）、MAX_CALLS 防护、LangSmith 追踪
  • 必做练习：不依赖 ToolNode，手写完整的 ReAct 循环，含错误处理和循环上限
  • 寓言语境：侦探的方法 — 思考→行动→观察→再思考
  • 配置 LangSmith：添加 LANGCHAIN_TRACING_V2=true 环境变量
  • 面试预热：Q43-Q46 — Agent Loop + Self-Reflection

    • 🏁 里程碑 1：能独立构建单 Agent 通关检查

  • 盲区自测：盲区题 Q1-Q5（基础篇）— 全部答对才算通过
  • 面试模拟：能不看文档回答 L3 面试题 Q36-Q46
  • 实战检验：独立写一个天气助手 Agent，包含 3 个工具 + 条件边 + 循环防护

    • 🟡 第二阶段：生产级 Agent 开发（L05-L08）

    目标：理解持久化、流式输出、人机协同、错误处理——让 Agent 能上线。

    L05 · Memory & Persistence 持久化

  • 核心概念：Checkpointer (MemorySaver→SqliteSaver→PostgresSaver)、thread_id、时间旅行、Store API
  • 必做练习：将 L04 的 Agent 加上 SqliteSaver，测试重启进程后恢复对话
  • 寓言语境：书记官的账本 — Checkpoint 是自动存档的账本
  • 面试预热：Q61-Q64 — Checkpointing + Time Travel
    • ↓

    L06 · Human-in-the-Loop 人机协同

  • 核心概念：interrupt()、Command(resume=...)、interrupt_before/after、审批工作流
  • 必做练习：给 Agent 加审批：发送邮件前暂停等待人类确认
  • 寓言语境：桥上的收费站 — interrupt 是桥上的检查站
  • 面试预热：Q67-Q68 — HITL 审批实现
    • ↓

    L07 · Streaming 流式输出

  • 核心概念：4 种 Streaming Mode (values/updates/messages/custom)、astream_events(v2/v3)、get_stream_writer
  • 必做练习：实现同时输出 token 流 + 节点状态 + 自定义进度事件的 Agent
  • 寓言语境：信使与碎片 — Streaming 是信使逐片送达
  • 面试预热：Q65-Q66 — Streaming 模式选型
    • ↓

    L08 · Error Handling 错误处理

  • 核心概念：4 层错误处理策略、RetryPolicy、fallback 节点、GRAPH_RECURSION_LIMIT
  • 必做练习：为 Agent 的每个工具调用添加重试+降级策略
  • 寓言语境：城墙上的四道门 — 每道门是一层防御
  • 面试预热：Q75-Q76 — 错误处理分级 + 重试设计

    • 🏁 里程碑 2：Agent 可以上线 通关检查

  • 盲区自测：盲区题 Q6-Q9（专业篇）— 全部答对才算通过
  • 面试模拟：能回答 L4 面试题 Q61-Q80 中的 80%
  • 实战检验：为一个 Agent 添加 SqliteSaver + 审批流 + token 级流式输出 + 错误重试

    • 🔴 第三阶段：架构师级 Agent 系统（L09-L12）

    目标：设计多 Agent 架构、并行执行、部署到生产、评估监控 —— 达到面试高级/架构师水平。

    L09 · Subgraphs 子图

  • 核心概念：子图复用、Shared State vs Isolated State、Command 跨图跳转
  • 必做练习：将一个复杂 Agent 拆分为父图+子图结构
  • 寓言语境：城中之城 — 子图是城内的独立城区
  • 面试预热：Q69-Q71 — Subgraph + Command 跳转
    • ↓

    L10 · Multi-Agent 多智能体

  • 核心概念：Supervisor 模式、Swarm/Handoff 模式、A2A 协议入门、Agent 通信模式
  • 必做练习：构建 Supervisor + Worker 双 Agent 系统
  • 寓言语境：蜂巢的议会 — 蜂后(Supervisor) + 工蜂(Workers)
  • 面试预热：Q86-Q90 — Multi-Agent 设计 + 框架对比
    • ↓

    L11 · Parallel & Send 并行

  • 核心概念：Super-Step、Send API (map-reduce 扇出)、并行 Reducer、并发控制
  • 必做练习：实现一个并行搜索 Agent：同时搜索 5 个来源，聚合结果
  • 寓言语境：将军的侦察兵 — Send 是派出多个侦察兵并行侦察
  • 面试预热：Q49 — Super-Step + Send API
    • ↓

    L12 · Production 生产部署

  • 核心概念：langgraph deploy、Postgres+Redis 配置、LangSmith 监控、安全 CVE 防御、成本优化
  • 必做练习：写一份 langgraph.json 配置 + 生产部署清单
  • 寓言语境：从打铁铺到兵工厂 — 从单机开发到规模生产
  • 面试预热：Q87-Q100 — 生产架构 + 安全 + 成本

    • 🏁 里程碑 3：能设计并部署生产级系统 最终通关

  • 盲区自测：盲区题 Q10-Q17（专家篇+2026前沿）— 全部答对
  • 面试模拟：能回答 L5 面试题 Q86-Q100 中的 80%
  • 最终实战：不翻文档，设计一个 Multi-Agent 客服系统（含架构图+State设计+部署方案）
  • 领域全貌：能讨论 领域全貌 中的共识、争议和开放问题

    • 🏆 精通最终验证
