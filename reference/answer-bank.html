A1 · Self-Attention 机制A1 · Self-Attention Mechanism

核心答案Core Answer

Self-Attention 的核心思想是：The core idea of Self-Attention is:让序列中的每个 token 都能直接"看到"序列中的所有其他 token，并根据相关性加权聚合信息allows each token in a sequence to directly "see" all other tokens and aggregate information weighted by relevance。公式为：. The formula is:

Attention(Q, K, V) = softmax(QK^T / √d_k) × V

直觉理解：Intuitive Understanding:

  • Q (Query)：当前 token 在"问"："哪些 token 和我相关？": The current token is "asking": "Which tokens are relevant to me?"
  • K (Key)：每个 token 在"答"："我是这种类型的 token": Each token is "answering": "I am this type of token"
  • V (Value)：每个 token 的"实际内容"，被提取和聚合: The "actual content" of each token, extracted and aggregated
  • QK^T：计算注意力分数（相关性矩阵）: Compute attention scores (relevance matrix)
  • softmax：归一化为概率分布: Normalize to a probability distribution
  • × V：按注意力权重加权聚合: Weighted aggregation by attention weights

"Attention Is All You Need" 的含义：Transformer 完全抛弃了 RNN/LSTM 的循环结构，仅用 Attention 机制就实现了序列建模，带来了三个革命性优势： means: Transformer completely abandoned the recurrent structure of RNN/LSTM, achieving sequence modeling using only the Attention mechanism, bringing three revolutionary advantages:

  1. 并行计算Parallel Computation：所有 token 同时处理，无需像 RNN 那样串行: All tokens are processed simultaneously, without the sequential dependency of RNNs
  2. 长距离依赖Long-range Dependencies：O(1) 的路径长度，任意两个 token 之间直接交互: O(1) path length, direct interaction between any two tokens
  3. 可解释性Interpretability：注意力权重可视化了模型"关注"的位置: Attention weights visualize where the model is "attending"

A2 · NumPy 实现 Scaled Dot-Product AttentionA2 · NumPy Implementation of Scaled Dot-Product Attention

标准实现Standard Implementation

import numpy as np

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q, K, V: (batch, n_heads, seq_len, d_k)
    mask: (batch, 1, seq_len, seq_len) or None
    """
    d_k = Q.shape[-1]

    # Step 1: 计算注意力分数
    scores = np.matmul(Q, K.transpose(0, 1, 3, 2))  # (batch, heads, seq_q, seq_k)

    # Step 2: 缩放 (关键！防止 softmax 落入梯度饱和区)
    scores = scores / np.sqrt(d_k)

    # Step 3: Masking (如 causal mask 或 padding mask)
    if mask is not None:
        scores = np.where(mask == 0, -1e9, scores)

    # Step 4: Softmax (沿最后一维)
    # 数值稳定版: exp(x - max(x))
    scores_max = np.max(scores, axis=-1, keepdims=True)
    scores_exp = np.exp(scores - scores_max)
    attention_weights = scores_exp / np.sum(scores_exp, axis=-1, keepdims=True)

    # Step 5: 加权聚合
    output = np.matmul(attention_weights, V)

    return output, attention_weights

  • 必须做Must use scores - scores_max 保证数值稳定性，直接用to ensure numerical stability; directly using np.exp(scores) 会在 scores 较大时溢出will overflow when scores are large
  • mask 中 padding 位置填In the mask, fill padding positions with -1e9 而非 而非 -np.inf，因为 softmax 遇到 NaN 会传播, because softmax propagates NaN

A3 · 为什么除以 √d_k？A3 · Why Divide by √d_k?

核心答案Core Answer

原因：防止点积方差随维度增大而增大，导致 softmax 梯度消失。Reason: Prevent dot-product variance from growing with dimensionality, which causes softmax gradient vanishing.

数学推导：Mathematical Derivation:

  • 假设 Q 和 K 的每个元素是独立同分布的，均值为 0，方差为 1Assume each element of Q and K is i.i.d. with mean 0 and variance 1
  • QK^T 是 d_k 个独立随机变量的点积和，其QK^T is the dot-product sum of d_k independent random variables, and its方差为 d_kvariance is d_k
  • 如果不缩放，当 d_k 很大时（如 64 或 128），QK^T 的值会很大Without scaling, when d_k is large (e.g., 64 or 128), QK^T values become very large
  • 大的值经过 softmax 后，分布变得极度尖锐（接近 one-hot）Large values after softmax make the distribution extremely peaked (near one-hot)
  • softmax 的梯度在尖锐分布下几乎为零 →The gradient of softmax under peaked distributions is near zero → 梯度消失Vanishing Gradient
  • 除以 √d_k 将方差缩放回 1，使 softmax 输出保持平滑，梯度可以有效传播Dividing by √d_k scales variance back to 1, keeping softmax output smooth so gradients can propagate effectively

如果是 d_k 而不是 √d_k？What if we divide by d_k instead of √d_k?除以 d_k 会导致方差变为 1/d_k，softmax 过于平滑（接近均匀分布），信息丢失。Dividing by d_k makes variance 1/d_k, causing softmax to be too smooth (near uniform), losing information.

PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

RoPE (Rotary Position Embedding)（LLaMA, Qwen, DeepSeek 等主流模型使用）：(used by mainstream models like LLaMA, Qwen, DeepSeek):

  • 核心思想Core Idea：通过旋转矩阵对 Q 和 K 进行编码，使得 QK^T 的结果自然包含相对位置信息: Encode Q and K via rotation matrices so that QK^T naturally contains relative position information
  • 数学本质Mathematical Essence：对 Q 和 K 的每对维度施加一个旋转，旋转角度与位置 m 成正比: Apply a rotation to each dimension pair of Q and K, with rotation angle proportional to position m

  2. 外推性好Good Extrapolation：配合 NTK-aware scaling 或 YaRN 可以很好地外推到更长序列: With NTK-aware scaling or YaRN, can extrapolate well to longer sequences

工作原理：工作原理：

  1. 传统 MHA：为每个 head 存储完整的 K 和 V 矩阵 → KV Cache 大小 = 2 × n_layers × d_model × seq_lenTraditional MHA: Store complete K and V matrices per head → KV Cache Size = 2 × n_layers × d_model × seq_len
