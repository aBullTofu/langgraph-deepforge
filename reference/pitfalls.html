🔥 七大反模式（来自社区论坛真实案例）

反模式 1：忘记 Reducer — 消息被覆盖

症状：消息列表只剩最后一条，前面的对话全部消失。

原因：State 字段默认"替换"而非"追加"。每个节点返回的新值覆盖旧值。

# ❌ 错误
class State(TypedDict):
    messages: list   # 每次写入覆盖上一次

# ✅ 正确
from langgraph.graph.message import add_messages
class State(TypedDict):
    messages: Annotated[list, add_messages]  # 追加而非覆盖

影响范围：L01-L04 的所有 State 定义。

反模式 2：ToolNode 输出覆盖对话导致 OpenAI 400

症状：messages with role 'tool' must be a response to a preceding message with 'tool_calls'

原因：ToolNode 返回的 ToolMessage 覆盖了对话历史，导致 LLM 收到的上下文里 tool 消息前面没有 assistant 消息。

# 这个错误的本质是反模式 1 在工具调用场景下的表现
# 解决：确保 messages 字段使用 add_messages reducer

影响范围：L03（工具调用）、L04（ReAct 循环）

反模式 3：突变 State（原地修改）

症状：不可预测的行为，状态在不同节点间"泄露"。

# ❌ 错误 — 原地修改
def bad_node(state):
    state["messages"].append(new_msg)  # 直接改了传入的 state！
    return state  # 返回完整 state

# ✅ 正确 — 返回部分更新
def good_node(state):
    return {"messages": [new_msg]}  # 只返回本节点负责的键

铁律：State 是只读的。节点只返回自己负责的键的更新 dict。

反模式 4：手动追加 + Reducer 双重追加

症状：消息列表中每条消息出现两次（重复）。

# ❌ 错误 — 手动追加了，reducer 又追加一次
def node(state):
    return {"messages": state["messages"] + [new_msg]}

# ✅ 正确 — 让 reducer 处理合并
def node(state):
    return {"messages": [new_msg]}

反模式 5：Optional 包裹顺序错误

# ❌ — reducer 被忽略
logs: Optional[Annotated[list[str], add]]

# ✅ — Annotated 必须在最外层
logs: Annotated[Optional[list[str]], add]

反模式 6：条件边 + 固定边混用导致并行

症状：条件边的分支全部执行，而非只执行一个。

原因：从同一个节点同时添加了固定边和条件边 → LangGraph 理解为"都要执行"。

# ❌ — 两个都加了，导致并行
builder.add_edge("classify", "handle_a")   # 删掉这行！
builder.add_conditional_edges("classify", router, mapping)

# ✅ — 只用条件边
builder.add_conditional_edges("classify", router, mapping)

反模式 7：没有 Checkpointer 就用 interrupt()

症状：interrupt() 不生效，图不暂停，或报错 MISSING_CHECKPOINTER。

# ❌ — 没有 checkpointer
graph = builder.compile()  # 缺少 checkpointer！
def node(state):
    interrupt("等等！")  # 不生效

# ✅ — 必须注入 checkpointer
from langgraph.checkpoint.memory import InMemorySaver
graph = builder.compile(checkpointer=InMemorySaver())

🔧 七种"静默退出"与排查

图停下来了但没有报错？逐一排查：

  1. 条件边返回了 END — 这是正常行为，检查路由逻辑是否符合预期
  2. 没有节点被调度 — 条件边映射字典缺少某个返回值 → 加 else 兜底
  3. 写入了 State 未声明的键 — LangGraph 静默忽略，后续节点读不到数据
  4. 显式连接到 END — 检查是否有不必要的 add_edge(xxx, END)
  5. 递归限制 — 默认最多递归 25 次，可在 compile 时设置 recursion_limit
  6. interrupt() 没有 checkpointer — 行为不可预测
  7. 节点返回了空 dict — 图认为无事可做
