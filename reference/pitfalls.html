🔥 七大反模式（来自社区论坛真实案例）
🔥 Seven Anti-Patterns (Real Cases from Community Forums)

反模式 1：忘记 Reducer — 消息被覆盖
Anti-Pattern 1: Forgetting the Reducer — Messages Get Overwritten

症状：
Symptoms:
消息列表只剩最后一条，前面的对话全部消失。
Only the last message remains in the list; all prior conversation is gone.

原因：
Cause:
State 字段默认"替换"而非"追加"。每个节点返回的新值覆盖旧值。
State fields default to "replace" rather than "append." Each node's returned value overwrites the previous one.

# ❌ Wrong
class State(TypedDict):
    messages: list   # Each write overwrites the previous one

# ✅ Correct
from langgraph.graph.message import add_messages
class State(TypedDict):
    messages: Annotated[list, add_messages]  # Append instead of overwrite

影响范围：
Impact Scope:
L01-L04 的所有 State 定义。
All State definitions in L01-L04.

反模式 2：ToolNode 输出覆盖对话导致 OpenAI 400
Anti-Pattern 2: ToolNode Output Overwrites Conversation, Causing OpenAI 400

症状：
Symptoms:
messages with role 'tool' must be a response to a preceding message with 'tool_calls'

原因：
Cause:
ToolNode 返回的 ToolMessage 覆盖了对话历史，导致 LLM 收到的上下文里 tool 消息前面没有 assistant 消息。
The ToolMessage returned by ToolNode overwrites conversation history, so the LLM receives a context where a tool message has no preceding assistant message.

# This error is essentially Anti-Pattern 1 in the tool-calling context
# Fix: ensure the messages field uses the add_messages reducer

影响范围：
Impact Scope:
L03（工具调用）、L04（ReAct 循环）
L03 (Tool Calling), L04 (ReAct Loop)

反模式 3：突变 State（原地修改）
Anti-Pattern 3: Mutating State (In-Place Modification)

症状：
Symptoms:
不可预测的行为，状态在不同节点间"泄露"。
Unpredictable behavior; state "leaks" between nodes.

# ❌ Wrong — in-place mutation
def bad_node(state):
    state["messages"].append(new_msg)  # Directly mutates the incoming state!
    return state  # Returns the full state

# ✅ Correct — return partial update
def good_node(state):
    return {"messages": [new_msg]}  # Only return the keys this node is responsible for

铁律：
Iron Rule:
State 是只读的。节点只返回自己负责的键的更新 dict。
State is read-only. Nodes only return an update dict for the keys they are responsible for.

反模式 4：手动追加 + Reducer 双重追加
Anti-Pattern 4: Manual Append + Reducer Double-Append

症状：
Symptoms:
消息列表中每条消息出现两次（重复）。
Each message appears twice (duplicated) in the message list.

# ❌ Wrong — manually appended, then reducer appends again
def node(state):
    return {"messages": state["messages"] + [new_msg]}

# ✅ Correct — let the reducer handle merging
def node(state):
    return {"messages": [new_msg]}

反模式 5：Optional 包裹顺序错误
Anti-Pattern 5: Incorrect Optional Wrapping Order

# ❌ — reducer is ignored
logs: Optional[Annotated[list[str], add]]

# ✅ — Annotated must be the outermost wrapper
logs: Annotated[Optional[list[str]], add]

反模式 6：条件边 + 固定边混用导致并行
Anti-Pattern 6: Mixing Conditional and Fixed Edges Causes Parallel Execution

症状：
Symptoms:
条件边的分支全部执行，而非只执行一个。
All branches of the conditional edge execute, rather than just one.

原因：
Cause:
从同一个节点同时添加了固定边和条件边 → LangGraph 理解为"都要执行"。
Both a fixed edge and a conditional edge were added from the same node → LangGraph interprets this as "execute both."

# ❌ — both added, causing parallel execution
builder.add_edge("classify", "handle_a")   # Delete this line!
builder.add_conditional_edges("classify", router, mapping)

# ✅ — only use conditional edges
builder.add_conditional_edges("classify", router, mapping)

反模式 7：没有 Checkpointer 就用 interrupt()
Anti-Pattern 7: Using interrupt() Without a Checkpointer

症状：
Symptoms:
interrupt() 不生效，图不暂停，或报错 MISSING_CHECKPOINTER。
interrupt() has no effect, graph doesn't pause, or MISSING_CHECKPOINTER error.

# ❌ — no checkpointer
graph = builder.compile()  # Missing checkpointer!
def node(state):
    interrupt("Wait!")  # Has no effect

# ✅ — must inject checkpointer
from langgraph.checkpoint.memory import InMemorySaver
graph = builder.compile(checkpointer=InMemorySaver())

🔧 七种"静默退出"与排查
🔧 Seven Types of "Silent Exit" and Diagnosis

图停下来了但没有报错？逐一排查：
Graph stopped but no error? Check one by one:

  1. 条件边返回了 END
    Conditional edge returned END
     —
    这是正常行为，检查路由逻辑是否符合预期
    This is normal behavior; check whether the routing logic matches expectations
  2. 没有节点被调度
    No node was scheduled
     —
    条件边映射字典缺少某个返回值 → 加 else 兜底
    Conditional edge mapping dict is missing a return value → add an else fallback
  3. 写入了 State 未声明的键
    Wrote a key not declared in State
     —
    LangGraph 静默忽略，后续节点读不到数据
    LangGraph silently ignores it; downstream nodes can't read the data
  4. 显式连接到 END
    Explicitly connected to END
     —
    检查是否有不必要的 add_edge(xxx, END)
    Check for unnecessary add_edge(xxx, END)
  5. 递归限制
    Recursion limit
     —
    默认最多递归 25 次，可在 compile 时设置 recursion_limit
    Default max 25 recursions; set recursion_limit at compile time
  6. interrupt() 没有 checkpointer
    interrupt() without checkpointer
     —
    行为不可预测
    Behavior is unpredictable
  7. 节点返回了空 dict
    Node returned an empty dict
     —
    图认为无事可做
    Graph considers there is nothing to do
